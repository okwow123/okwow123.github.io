Flexibility of Inference Control

1
Ollama — "Environment-Level Control"
Control Unit
  • Server level
  • Model execution environment
What You Can Control
  • OLLAMA_NUM_PARALLEL
  • num_ctx
  • Model selection
  • Quantization options
Characteristics
  • Configuration is done before inference starts
  • No intervention during inference
  • No direct control over token generation
Abstraction Level
  • Highest level (close to a black box)
  • Operates at the level of "running a model"
Advantages
  • Simple to use
  • Good for local experimentation
  • Fast deployment
Limitations
  • No conditional logic control
  • Difficult to orchestrate multi-step reasoning
  • No token-level control
2
vLLM — "Request-Level Control"
Control Unit
  • API request level
What You Can Control
  • temperature
  • top_p
  • max_tokens
  • stop sequences
  • Beam search
Characteristics
  • Configurable per request
  • Stateless architecture
  • No intervention once inference has started
Abstraction Level
  • Medium level
  • Controls how each request is generated
Advantages
  • High-performance serving
  • Efficient batching
  • OpenAI-compatible API
  • Suitable for large-scale services
Limitations
  • Cannot program the reasoning path
  • Difficult to share state across requests
  • No branching during token generation
3
SGLang — "Inference Algorithm-Level Control"
Control Unit
  • Token generation step
  • Reasoning path
What Is Possible
  • Branching with Python conditionals after gen()
  • fork() → parallel reasoning paths
  • select() → enforce model selection
  • regex → constrain output format
  • Dynamic reasoning orchestration
Characteristics
  • Inference can be written like a program
  • Enables multi-agent style compositions
  • Real-time control over generation
Abstraction Level
  • Lowest level (most flexible)
  • You don't just call the LLM — you control it
Advantages
  • Full programmatic control over reasoning
  • Supports dynamic branching and forking
  • Powerful multi-agent orchestration
Limitations
  • Higher learning curve
  • More complex setup than Ollama/vLLM

Structural Comparison

Category Ollama vLLM SGLang
